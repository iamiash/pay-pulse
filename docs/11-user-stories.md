User Stories

These user stories define the development scope for the PayPulse tracking application based on user requirements.

### Core Dashboard & Management
User Story 1: As a busy student, I want to manually type in a new subscription with its price and due date so that I can keep all my tech expenses in one central database.
User Story 2: As a trial user, I want to easily delete a subscription from my tracking list once I have canceled the service so that my active dashboard stays clean and accurate.

### Alerts & Aggregations
User Story 3: As someone tracking free trials, I want the app to show a clear red or yellow badge next to items expiring soon so that I can cancel them before getting auto-charged.
User Story 4: As a budget-conscious user, I want to see a running total of my monthly spending at the top of the screen so that I know exactly how much money I am spending on digital tools.
